状態機械（FSM）エディタ兼シミュレータ。3ファイル構成です。

egui/eframe 

main.rs  起動・ウィンドウ設定
app.rs   GUI・編集/実行UI
fsm.rs   データモデル・実行エンジン

エントリポイント
eframe::NativeOptions でウィンドウ
タイトル「状態機械」、1024×768）を設定し FsmApp を起動するだけ。

ドメインモデル＋実行エンジン（UI非依存）

データ構造
（serde で TOML 保存/読込）
State … id / label / 座標(x,y) / initial(初期状態) / accepting(受理状態)
Transition … from→to / event（空ならオート遷移）/ actions（順次実行するアクション列）
Action … Wait（待機・ms/sec）/ Message（メッセージ表示）/ SetVar（変数代入）。{名前} で変数展開
Variable … コンテクスト変数の初期値
Fsm … states/transitions/variables を束ねる。add_*/remove_*、id検索、TOML入出力

実行系
状態機械を60Hz固定タイムステップで駆動。
current_state / vars（実行中変数）/ log / message を保持
tick() … 待機の経過処理→アクション消化→遷移完了で次状態へ
trigger() … 名前付きイベントの手動発火（分岐用）
try_start_auto() … オート遷移（イベント名空）の自動発火
interpolate() … {変数名} を現在値に置換
Playback（private）… 再生中遷移の進捗（どのアクション／待機残り秒）

app.rs
GUI 層（FsmApp）

2モードを持つ
編集モード … キャンバスに状態を円で描画、遷移を矢印で描画
ツール: 選択 / 状態追加(タッチ) / 遷移追加（ドラッグ）。状態のドラッグ移動、Delete/Backspace削除
右ペイン: 選択中の状態/遷移のプロパティ（ラベル・初期/受理・イベント名・アクション編集）
左ペイン: コンテクスト変数の初期値編集
実行モード … Runner
を60Hzで回し、現在状態を緑ハイライト・メッセージ表示・イベント送信・ログ・実行中変数の表示
その他: 日本語フォント読込（setup_fonts）、TOMLの開く/保存（⌘O/⌘S）、矢印描画ヘルパ
  
所感

役割分離はきれい: fsm.rs（ロジック）と app.rs（描画）が分かれており、
Runner がUIに依存していないのは良い設計です。
app.rs が722行とやや肥大: 描画（draw_canvas）・プロパティ編集（draw_properties）・実行パネル（draw_run_
  panel）で機能が明確なので、もし整理したければ ui/
  モジュールへの分割余地があります。今は3ファイルで見通せる範囲です。
