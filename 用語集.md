# 用語集

状態機械エディタで使う用語と、対応するコード上の識別子をまとめる。

## ドメイン用語（状態機械の概念）

| 用語 | コード | 意味 |
|------|--------|------|
| 状態機械 / FSM | `Fsm` | 状態・遷移・変数の集合。エディタが編集する対象全体。 |
| 状態 | `State` | 図中の円。`label`（表示名）と座標 `x, y` を持つ。 |
| 初期状態 | `State.initial` | 実行開始時に最初に入る状態。状態を最初に追加すると自動で `true`。左向き矢印で表示。 |
| 受理状態 | `State.accepting` | 受理（終了）を表す状態。二重円で表示。実行ロジック自体には影響しない表示上の印。 |
| 遷移 | `Transition` | 状態から状態への矢印（`from` → `to`）。発火すると `actions` を順次実行して `to` へ移る。 |
| イベント | `Transition.event` | 遷移を起こすきっかけの名前。空文字なら「オート遷移」。 |
| オート遷移 | `event` が空 | イベント名なしの遷移。状態に入ると自動で発火する。図では `(auto)` と表示。 |
| 名前付きイベント | `event` に文字列 | 実行モードで手動送信して発火させる遷移。分岐に使う。 |
| アクション | `Action` | 遷移発火時に上から順に実行される処理。待機・メッセージ・変数設定の3種。 |
| 待機 | `Action::Wait` | 指定時間だけ次のアクションを止める。単位はミリ秒/秒（`TimeUnit`）。 |
| メッセージ | `Action::Message` | 実行画面に文字列を表示する。`{変数名}` を展開できる。 |
| 変数設定 | `Action::SetVar` | 変数に値を代入する。値の中の `{変数名}` を展開できる。 |
| コンテクスト変数 | `Variable` | 名前と値の組。編集時に初期値を設定し、実行中に読み書きされる。 |
| 変数展開 / 補間 | `Runner::interpolate` | テキスト中の `{名前}` を現在の変数値に置き換える処理。 |

## 実行（シミュレーション）用語

| 用語 | コード | 意味 |
|------|--------|------|
| ランナー | `Runner` | 状態機械を 60Hz で動かす実行エンジン。UI に依存しない。 |
| 固定タイムステップ | `step = 1/60` | 1フレームを 1/60 秒として一定間隔で進める駆動方式。 |
| アキュムレータ | `run_accumulator` | 実フレーム間隔の端数をためて、固定ステップに小分けするための変数。 |
| ティック | `Runner::tick` | 1ステップ（dt 秒）進める処理。待機消化・アクション実行・状態遷移を行う。 |
| 再生 / プレイバック | `Playback` | 現在発火中の遷移の進捗。どのアクションまで進んだか・待機残り時間を保持。 |
| 発火 / トリガ | `Runner::trigger` | 名前付きイベントを送って対応する遷移を開始すること。 |
| 現在状態 | `Runner.current_state` | 実行中に今いる状態。図では緑でハイライト。 |
| 実行ログ | `Runner.log` | 遷移・メッセージ・変数変更の履歴。実行パネルに新しい順で表示。 |

## エディタ（GUI）用語

| 用語 | コード | 意味 |
|------|--------|------|
| アプリ本体 | `FsmApp` | egui アプリ。編集/実行の状態を持ち、毎フレーム UI を描く。 |
| モード | `Mode` | `Edit`（編集）と `Run`（実行）の切り替え。 |
| ツール | `Tool` | 編集モードの操作モード。選択 / 状態追加(タッチ) / 遷移追加。 |
| 選択 | `Selection` | 今選んでいる対象。なし / 状態 / 遷移 のいずれか。 |
| キャンバス | `draw_canvas` | 状態と遷移を描く中央の作図領域。 |
| プロパティパネル | `draw_properties` | 右側。選択中の状態/遷移の属性を編集する。 |
| コンテクストパネル | `draw_context` | 左側。変数の初期値（編集時）や実行中の値（実行時）を表示。 |
| 実行パネル | `draw_run_panel` | 実行モードの右側。現在状態・メッセージ・イベント送信・ログを表示。 |

## ファイル / 永続化

| 用語 | コード | 意味 |
|------|--------|------|
| シリアライズ | `serde` / `to_toml` | 状態機械を保存用テキスト（TOML）に変換すること。 |
| TOML 入出力 | `Fsm::to_toml` / `from_toml` | 状態機械を `.toml` で保存・読込する。 |
